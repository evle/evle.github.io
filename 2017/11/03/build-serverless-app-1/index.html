Building Serverless Web Application 1

Publish on 2017-12-15

Why I prefer Serverless

If you want to validate an idea through a Minimum Viable Product (MVP) without losing time to market, consider using serverless to save time.

Thinking Serverless

A serverless solution is entirely event-driven. 每当用户request信息的时候, a triger 将会通知云供应商去找到并执行你的代码去回应用户的请求. 在serverless架构中, 不需要加载整个应用程序的代码，通常只需要加载一些函数(通常是压缩过的包)去回应用户的请求.

Serverless model

  1. The user sends a request to an address handled by the cloud provider.
  2. Based on the message, the cloud service tries to locate which package must be used to answer the request.
  3. The package (or function) is selected and loaded into a Docker container.
  4. The container is executed and outputs an answer.
  5. The answer is sent to the original user.

Severless & PaaS

PaaS和Serverless很相似, 但在PaaS中代码会一直运行并监听客户端的请求，无论是否有用户.

Using the AWS

  1. Install the AWS SDK
    npm install aws-sdk --save
  2. Setting credentials

`

~/.bash_profile

export AWS_ACCESS_KEY_ID=YOUR-KEY
export AWS_SECRET_ACCESS_KEY=YOUR-KEY
export AWS_REGION=YOUR-REGION
`

  1. Creating node file
    Amazon Simple Storage Service (S3) is a service where you can save any type of file such as images, log files, and backups.
    Amazon calls each file as an object and, to store files, you need a root folder that is called as a bucket. We will take advantage of S3 as it is a very cheap storage service and it has the flexibility to share files to build our low cost and serverless frontend. In our bucket, we’ll add all static files of our frontend, including HTML, CSS, JavaScript, images, and so on.
    `
    const AWS = require(‘aws-sdk’);

     const s3 = new AWS.S3();

 s3.listBuckets((err, data) => {
   if (err) console.log(err, err.stack); // an error occurred
   else console.log(data.Buckets); // successful response
 });

    `
