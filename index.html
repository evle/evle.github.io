Markdown简明指南

什么是Markdown?

Markdown is a text-to-HTML conversion tool for web writers. Markdown allows you to write using an easy-to-read,
easy-to-write plain text format, then convert it to structurally valid XHTML (or HTML).

为什么使用Markdown?

直观 简单 便捷, 使你更加专注于文档的内容而不是样式

表格

| Name | age |
| ---- | ---:|
| Max  | 24  |

单行代码

`code`

多行代码

```code```

分割线

---

标题

# 一级标题 #
## 二级标题 ##
### 三级标题 ###

列表

  2. 语法中不要包含没有意义的空格

使用GitHub+Hexo搭建个人博客2

介绍

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

安装

在安装Hexo之前需要先安装以下软件

  • Node.js
  • Git (注意需要添加Git的bin目录至环境变量)
    最后执行以下命令安装Hexo

使用流程

  1. 首先执行以下命令初始化一个Hexo工程.

    hexo init
    cd
    npm install

  2. 在新建的HexoProject文件夹中使用以下命令创建文章, 然后用你喜欢的编辑器打开所创建的文章进行书写

    hexo new

  3. 写好文章后需要生成网页

    hexo generate # 可简化为 hexo g

  4. 本地预览
    启动Hexo服务器

    hexo server # 启动后登陆localhost:4000进行预览

布署

布署是将你本地Hexo工程所生成的页面布署到github的一个过程.

  1. 安装hexo布署模块

    npm install hexo-deployer-git –save

  2. 在_config.yml文件中修改一些设置

    deploy:

    type: git
repo: <repository url>    # e.g., repo: https://github.com/evle/evle.github.io.git

  3. 生成页面并布署至github

    hexo deploy –generate # 可简化为 hexo d -g

Tips: 当修改_config.yml文件并且布署后发现网页没有更新时

rm -rf .deploy_git/
hexo clean
hexo d -g

更换主题

  1. 从https://hexo.io/themes/ 挑选自己心仪的主题并下载

  2. 将下载的主题放至Hexo工程目录中的themes目录

  3. 在_config.yml中修改theme的名字为新主题的名字

If you want to read more details about Hexo, https://hexo.io/docs/

使用GitHub+Hexo搭建个人博客(一)

写技术博客对于程序员来说是一个很好的习惯

  1. Learning by teaching
    能否给一个不懂的人讲清来一个知识点的来龙去脉是一种很好的衡量我们理解程度的方法, 而且被教者的提问有时也会使我们引发新的思考.

  2. Solution record
    记录, 积累问题的解决方案.

  3. Think better
    书写是一个帮助思考的过程, 当你把想法真正写下来的时候也许你会发现有些想法特别理想化.

Let’s start.

  1. 首先你需要拥有一个github账号(点击注册)

  2. 登陆github账号之后新建一个Repository, Repository name需要按照固定格式username.github.io(e.g., evle.github.io)

  3. 在刚才创建好的仓库页面点击Setting, 下拉到底部点击Automatic page generator, 现在你的github博客已经搭建成功并且可以使用username.github.io进行访问

  4. 绑定域名(可选)
    username.github.io仓库下新建一个名为CNAME的文件, 文件内容填写想要绑定的域名(e.g., evleblog.com)
    以GoDaddy域名为例, 登陆GoDaddy账户之后点击域名右侧的管理, 然后在域名设置选项中点击管理DNS之后做如下设置:

  • 在A(Host)中设置指向地址为192.30.252.153
  • 在CName中设置www指向username.github.io(e.g., evle.github.io)
    保存设置之后等待一段时间即可使用域名访问github博客
