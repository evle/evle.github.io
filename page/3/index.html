koa2

Koa is a new web framework designed by the team behind Express, which aims to be a smaller, more expressive, and more robust foundation for web applications and APIs. Through leveraging generators Koa allows you to ditch callbacks and greatly increase error-handling. Koa does not bundle any middleware within core, and provides an elegant suite of methods that make writing servers fast and enjoyable.

环境搭建

  1. 新建一个目录并在目录中新建 app.js

  2. 在目录中新建 package.js 并添加依赖包

    添加完成之后，在当前目录下使用 npm instll 命令安装所添加的依赖包

  3. 新建自动转码文件 start.js

  4. 在当前目录下执行 node start 命令后访问 localhost:3000 ，如果显示 Hello, koa2! 则环境配置成功.

Koa2 使用流程

Koa2的使用流程如下：

  1. Import ‘koa’ class
  2. Create a Koa object;
  3. Request handing
  4. Listen on a specific port

每当监听的端口收到一个http请求，Koa2就会调用通过 app.use()注册的 async 函数，并传入 ctx 和 next参数。ctx 可以设置返回内容比如ctx.response.body ＝ ‘<p> unless you show me how’. next 参数用来调用下一个 async 函数。

Middleware

async 函数就是middleware. 通过 app.use() 可以注册middleware并且 app.use() 的调用顺序决定了middleware的执行顺序, 使用 await next() 调用下一个middleware. 通过middleware的组合实现各种功能是Koa2的精髓。

单元测试

预计完成时间 2016－12-30之前

单元测试用来测试函数，类或者模块。编写单元测试可以提升我们的编码效率，尤其在经常改动的项目中。当你编写完一个模块，你需要花费时间去测试自己的模块是否有Bug。但当代码有改动，你还需要再次花费时间去测试代码，这无疑降低了开发效率。如果使用单元测试， 只需要编写一次测试代码 ，以后每当代码改动时即可用已编写的单元测试来测试改动后的代码是否有Bug。

使用Atom+Markdown记录笔记

相比Sublime Text3, Atom的Markdown插件提供了实时预览功能可以使我们记录笔记更加高效.

  1. Download Atom

  2. Install Markdown Plugins
    打开Atom，按下 Shift + cmd + p 后输入 install package 回车,搜索 Markdown 后安装 Markdown Preview Plus，Markdown-writer 和 Markdown toc.

安装好了以上工具之后, 我们就可以使用Atom+Markdown来记录笔记了.

  1. 打开Atom并新建一个以 md 格式结尾的文件
  2. 使用Markdown语法记录笔记
  3. 使用 ctrl + shift + m 实时预览笔记内容, 如下图

Tips：
插入目录： Ctrl + Alt + C
插入链接： Cmd + Shift + K
插入图片： Cmd + Shift + I
更多快捷键： Packages -> Markdown Writer -> Open Cheat Sheet
自动补全：img+ tab

使用Atom推送笔记至博客

通过安装 terminal-plus 插件, 我们可以在笔记记录完成后直接在Atom中上传笔记到博客中.

  1. 安装 terminal-plus 插件
    terminal-plus 在最新版本的Atom中有Bug，所以我们需要手动安装 terminal-plus 并修改一下配置文件.

    • 打开终端并输入 apm install LarsKumbier/terminal-plus
    • 打开 ~/.atom/packages/terminal-plus/styles/themes/standard.less 并修改background-color: transparent;

  2. 在Atom中按下 Shift + cmd + t 调用终端，在终端中输入Hexo d -g

草稿功能

在编写长篇文章或者不能及时完成的文章时，可以使用 hexo new draft filename 命令来将文章新建在 _drafts 目录中，_drafts 目录中的草稿文件不会在部署时被上传到Blog. 当文章编写完时, 可以使用 hexo publish draft filename 命令将 _drafts 目录中的文件转移到 _posts 目录中并且进行部署。

添加404页面

在 source 目录下创建一个 404.html 页面，然后通过hexo g -d直接生成页面。404页面只对于绑定了顶级域名的博客有效。

CNAME 存放位置

对于绑定了顶级域名的博客，CNAME文件应该存放在 source 目录中。

使用GitHub+Hexo搭建个人博客(二)

预计完成时间 2016－12-30之前

介绍

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

安装

在安装Hexo之前需要先安装以下软件

  • Node.js

  • Git (注意需要添加Git的bin目录至环境变量)
    最后执行以下命令安装Hexo

    npm install -g hexo-cli

使用流程

  1. 首先执行以下命令初始化一个Hexo工程.

    hexo init <folder>
cd <folder>
npm install

  2. 在新建的HexoProject文件夹中使用以下命令创建文章, 然后用你喜欢的编辑器打开所创建的文章进行书写

    hexo new <title>

  3. 写好文章后需要生成网页

    hexo generate  # 可简化为 `hexo g`

  4. 本地预览
    启动Hexo服务器

    hexo server     # 启动后登陆`localhost:4000`进行预览

布署

布署是将你本地Hexo工程所生成的页面布署到github的一个过程.

  1. 安装hexo布署模块

    npm install hexo-deployer-git --save

  2. 在_config.yml文件中修改一些设置

    deploy:
    type: git
    repo: <repository url>    # e.g., repo: https://github.com/evle/evle.github.io.git

  3. 生成页面并布署至github

    hexo deploy --generate      # 可简化为 `hexo d -g`

Tips: 当修改_config.yml文件并且布署后发现网页没有更新时

rm -rf .deploy_git/
hexo clean
hexo d -g

更换主题

  1. 从https://hexo.io/themes/ 挑选自己心仪的主题并下载

  2. 将下载的主题放至Hexo工程目录中的themes目录

  3. 在_config.yml中修改theme的名字为新主题的名字

If you want to read more details about Hexo, https://hexo.io/docs/

使用GitHub+Hexo搭建个人博客(一)

预计完成时间 2016－12-30之前

写技术博客对于程序员来说是一个很好的习惯

  1. Learning by teaching
    能否给一个不懂的人讲清来一个知识点的来龙去脉是一种很好的衡量我们理解程度的方法, 而且被教者的提问有时也会使我们引发新的思考.

  2. Solution record
    记录, 积累问题的解决方案.

  3. Think better
    书写是一个帮助思考的过程, 当你把想法真正写下来的时候也许你会发现有些想法特别理想化.

Let’s start.

  1. 首先你需要拥有一个github账号(点击注册)

  2. 登陆github账号之后新建一个Repository, Repository name需要按照固定格式username.github.io(e.g., evle.github.io)

  3. 在刚才创建好的仓库页面点击Setting, 下拉到底部点击Automatic page generator, 现在你的github博客已经搭建成功并且可以使用username.github.io进行访问

  4. 绑定域名(可选)
    username.github.io仓库下新建一个名为CNAME的文件, 文件内容填写想要绑定的域名(e.g., evleblog.com)
    以GoDaddy域名为例, 登陆GoDaddy账户之后点击域名右侧的管理, 然后在域名设置选项中点击管理DNS之后做如下设置:

  • 在A(Host)中设置指向地址为192.30.252.153
  • 在CName中设置www指向username.github.io(e.g., evle.github.io)
    保存设置之后等待一段时间即可使用域名访问github博客
